# book-face-swap 

## 1. О проекте

Сервис берёт фото лица клиента и подставляет его в заранее подготовленные
иллюстрации-заготовки из книг. 
Клиент загружает фото → выбирает книгу и сцену → получает иллюстрацию, где персонаж нарисован с его чертами лица и
причёской, в оригинальной манере письма (мазки, палитра).

Заготовки статичны: детекция лица персонажа и маска головы считаются один
раз на сцену и кэшируются; при обслуживании клиента выполняются только коллаж и генерация.

## 2. Как работает пайплайн

```
иллюстрация ──► [0] RetinaFace: bbox+landmarks лица персонажа   (кэш: data/detections/)
              ──► [1] SAM2: маска "лицо+волосы" персонажа        (кэш: data/masks/)

фото клиента ─► [2] КОЛЛАЖ (hair_collage.py)
                    • face-parsing (SegFormer): маски волос и лица клиента
                    • InsightFace: 5 точек лица клиента и персонажа →
                      similarity-выравнивание головы (масштаб+поворот+сдвиг)
                    • волосы персонажа затираются (cv2.inpaint Telea),
                      живописное ЛИЦО персонажа сохраняется как style-reference
                    • волосы клиента (фото-пиксели) вклеиваются поверх
              ──► [3] ИНПЕЙНТ (output_inpainting_v3.py)
                    • SDXL-чекпоинт + ControlNet Tile (держит геометрию
                      коллажа: форму волос, пропорции головы)
                    • IP-Adapter FaceID PlusV2 (ArcFace + CLIP-эмбеддинг
                      лица клиента → сходство)
                    • промпт с авто-описанием клиента ("35 year old man" —
                      пол/возраст из InsightFace)
                    • N сидов → N кандидатов;
              ──► [4] ЗАЛИВКА ДЫР В МАСКЕ (background_fill.py)
                    • если новая голова меньше старой, внутри старого контура
                      остаётся кольцо псевдо-фона; face-parsing защищает новую
                      голову/шею/одежду, кольцо заливается одним доп.
                      SDXL-inpaint проходом, продолжающим фон
              ──► [5] ВЫБОР
                    • каждый кандидат оценивается ArcFace-сходством с
                      клиентом; best.png, grid.jpg, result.json
```

## 3. Установка и требования

**Железо.** NVIDIA GPU c ≥16 GB VRAM; целевая карта — **RTX 3090 (24 GB)**
на RunPod (при ≥18 GB VRAM весь пайплайн живёт на CUDA без cpu-offload).
Диск: ~15 GB под веса моделей (докачиваются при первом запуске).

**Скорость.** На RTX 3090 генерация одного изображения-кандидата занимает
примерно **8 секунд** (при уже загруженных моделях). Первый запуск дольше:
скачивание весов + загрузка пайплайна в память (~1–2 мин).


```bash
# 1) PyTorch под вашу CUDA (на RunPod-образах обычно уже установлен)
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu121

# 2) остальные зависимости
pip install -r requirements.txt
```

Структура данных:

```
data/
  illustrations/   # заготовки-иллюстрации (сцены книг)
  clients/         # фото клиентов (временные! см. retention policy)
  detections/      # кэш шага 0 (JSON + визуализация)
  masks/           # кэш шага 1 (маска + визуализация + JSON)
  outputs/         # результаты: <сцена>__<клиент>/candidate_*.png, best.png, grid.jpg
```

## 4. Запуск

### Весь пайплайн одной командой (рекомендуется)

```bash
python pipeline.py --image data/illustrations/spread_01.png \
                   --client data/clients/ivan.jpg \
                   --seeds 5 
```

Полезные флаги: `--likeness` (сходство важнее стиля),  
`--force-mask` (пересчитать маску),
`--no-bg-fill` (без заливки маски фоном), 

### Каждая часть отдельно

**Шаг 0 — детекция лица на иллюстрации** (RetinaFace):

```bash
python illustrations_detect.py                                # выбор из списка
python illustrations_detect.py --image data/illustrations/spread_01.png
```

Выход: `data/detections/<name>.json` + `<name>_vis.jpg` для проверки глазами.

**Шаг 1 — маска лицо+волосы** (SAM2; требует посчитанной детекции):

```bash
python illustrations_mask.py --image data/illustrations/spread_01.png
# тонкая настройка захвата волос:
python illustrations_mask.py --image ... --pad-up 1.0 --hair-points 3
```

Выход: `data/masks/<name>_mask.png` + визуализация + JSON. Обязательно
проверьте `_vis.jpg`: маска должна накрывать всё лицо и ВСЕ волосы персонажа.

**Шаг 2 — коллаж отдельно** (отладка, без диффузии):

```bash
python hair_collage.py --image data/illustrations/spread_01.png \
                       --client data/clients/ivan.jpg
```

Выход: `data/outputs/_collage_debug/` — коллаж, маска волос, регион инпейнта.

**Шаги 2–5 — генерация** (требует посчитанной маски):

```bash
python output_inpainting_v3.py --image data/illustrations/spread_01.png \
                               --client data/clients/ivan.jpg \
                               
```

## 5. Параметры генерации: что на что влияет

Дефолты — в `InpaintConfig` (`output_inpainting_v3.py`); 
### Главные параметры качества

| Параметр (CLI / поле) | Default | На что влияет |
|---|---|---|
| `--ip-scale` / `ip_scale` | 0.75 | Сила переноса личности клиента (IP-Adapter FaceID). Больше (→1.0) = похожее на клиента, но лицо «фотографичнее» и может спорить со стилем. Меньше = стильнее, но менее узнаваемо. Рабочий диапазон 0.6–1.0. |
| `--control-scale` / `control_scale` | 0.2 | Вес ControlNet Tile. Больше (0.35–0.5) = точнее держит форму волос из коллажа и пропорции головы, лечит «лицо слилось с фоном»; но фактура становится более фотографичной. Меньше = свободнее, живописнее. |
| `--strength` / `strength` | 0.6 | Насколько сильно перерисовывается зона. 1.0 = полностью с нуля (максимум свободы модели), 0.4–0.6 = ближе к коллажу (вклеенные волосы клиента сохраняют цвет/форму). |
| `--guidance` / `guidance` | 4.5 | Следование промпту. 4–6 норм; выше — «пережаренные» цвета. |
| `--steps` / `steps` | 35 | Шаги диффузии. Больше = чуть качественнее и медленнее; 30–40 достаточно. |
| `--seeds` / `seeds` | 5 | Сколько кандидатов. Больше = выше шанс удачного, линейно дольше. |
| `--likeness` | выкл | Пресет «сходство важнее стиля»: strength=1.0, ip_scale=1.0, control_scale=0.18, реалистичный промпт. Лицо максимально похоже, но может выбиваться из стиля. |

### Швы, нимб, вклейка

| Параметр | Default | На что влияет |
|---|---|---|
| `seam_dilate` | 48 | Запас маски **генерации** (px в кропе gen_size). Модели нужен запас на швы; слишком мало — видимая граница. |
| `--paste-dilate` / `paste_dilate` | 30 | Запас маски **вклейки** в оригинал. Меньше = нет «нимба» вокруг головы; больше = мягче шов, но ореол возвращается. |
| `mask_blur` | 8 | Размытие краёв масок. Больше = мягче переход, но «мыло» по границе. |
| `--no-bg-fill` / `bg_fill` | вкл | Шаг 4: заливка кольца псевдо-фона внутри старого контура головы. Отключайте только для отладки. |
| `halo_grow_px` (BackgroundFiller) | 12 | Насколько расширять найденное кольцо нимба перед заливкой. |

### Модели

| Параметр | Default | На что влияет |
|---|---|---|
| `--base-model` / `base_model` | `SG161222/RealVisXL_V5.0` | SDXL-чекпоинт. Фотореалистичные (RealVisXL, Juggernaut-XL) рисуют структуру лица лучше, но тянут в фотореализм; `diffusers/stable-diffusion-xl-1.0-inpainting-0.1` — «родной» inpaint-чекпоинт. Проверять глазами на своём арт-стиле. |
| `--faceid` / `faceid_version` | `plusv2` | `plusv2` = ArcFace + CLIP-ветка: лучше переносит возраст/пол/пропорции. `v1` — только ArcFace (хуже, для сравнения). |
| `faceid_lora_scale` | 0.9 | Вес вспомогательной LoRA FaceID (0.5–1.0). |
